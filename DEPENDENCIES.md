# OpenCurtainLab Dependencies

This list describes the software and runtime dependencies needed to build, flash, and use OpenCurtainLab.

## Firmware build

| Dependency | Used for | Notes |
|---|---|---|
| Arduino IDE or Arduino CLI | Compiling and uploading the ESP32 sketch | Board package must support ESP32 Arduino core APIs. |
| ESP32 Arduino core | WiFi, WebServer, HTTPClient, Preferences, mDNS, ADC, GPIO, timers | Provides `WiFi.h`, `WebServer.h`, `HTTPClient.h`, `WiFiClientSecure.h`, `Preferences.h`, `DNSServer.h`, `ESPmDNS.h`, `esp_timer_get_time()`. |
| ArduinoJson | JSON parsing and serialization | Current source still uses ArduinoJson v6-style APIs such as `StaticJsonDocument` and `DynamicJsonDocument`. |
| Adafruit GFX Library | OLED graphics base layer | Included by `src/DisplayManager.h`. |
| Adafruit SSD1306 | OLED display driver | Included by `src/DisplayManager.h`. |
| Wire | I2C transport for OLED | Bundled with Arduino/ESP32 core. |

## Firmware hardware expectations

| Item | Expected behavior |
|---|---|
| ESP32 board | 12-bit ADC available on configured sensor pins. |
| 128x64 SSD1306 OLED | I2C address `0x3C` by default. |
| Five phototransistor channels | Dark/raw idle near `4095`; incoming light pulls ADC raw value downward. |
| Flash sync input | Active-low contact on `PIN_FLASH_SENSOR` with internal pull-up. |
| Three buttons | Active-low buttons with firmware debounce. |

## WebUI runtime

| Dependency | Used for | Notes |
|---|---|---|
| Modern browser | Standalone WebUI execution | Uses standard HTML, CSS, JavaScript, `fetch`, `localStorage`, SVG, Blob downloads, and file import. |
| Browser support for `AbortSignal.timeout()` | API timeouts | Recent Chromium, Firefox, and Safari versions support it. Older browsers may need a small fallback if required. |
| Local network access to ESP32 | Device API | Browser must be able to reach `http://opencurtainlab.local` or the device IP. |

The WebUI has no npm runtime dependency. The compiled WebUI is a standalone HTML file generated from files in `web/`.

## Development and release tools

| Dependency | Used for | Notes |
|---|---|---|
| Python 3.10+ | Release and build scripts | Scripts use only the Python standard library. |
| Node.js | Optional JavaScript syntax check | Used with `node --check`; not required for the WebUI build itself. |
| GitHub Raw availability | WebUI download through firmware | `GET /` fetches `web/manifest.json` and then the selected compiled HTML URL from the manifest. |

## Generated artifacts

| File | Generated by |
|---|---|
| `src/SetupPortalHtml.h` | `tools/release.py` (`tools/build_setup_portal.py` remains a compatibility wrapper) |
| `web/compiled/compiled-v*.html` | `tools/release.py` (`tools/build_webui.py` remains a compatibility wrapper) |

Use `python3 tools/release.py` before publishing a release. It updates version references, rebuilds generated artifacts, and validates that the manifest contains a complete release entry.
